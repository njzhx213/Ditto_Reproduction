# Figures — what each one shows

Every figure here is generated by a script in `sim/` (performance model) or by
`sim/plot_rtl_fmax.py` (RTL). Numbers are labeled **paper-stated / measured / inferred /
swept**; nothing is tuned to land on a paper figure. Where a paper value is shown it is
an overlaid reference line, not a fitted point. Full derivations are in
[SUMMARY.md](SUMMARY.md); this file is a one-stop guide to reading the plots.

## Performance-model figures

### `fig5_bitwidth_sdm.png` — temporal-difference bit-width (paper Fig 5)
Stacked bars splitting SDM's temporal differences into **zero / 4-bit / >4-bit**, for
three columns: *Ours (dynamic)* 46% zero, *Ours (calib 99.9)* 27% zero, and *Paper SDM*
44%. Our dynamic-scale result (45.9% zero, the headline) sits next to the paper's 44.48%
— reproduced. The second "calib" column is shown deliberately: with a calibrated (not
per-tensor-max) scale the >4-bit fraction rises from ~0.7% toward the paper's ~4%,
making explicit that the >4-bit rate is **scale-dependent**, not a discrepancy. The
quantizer is the Fig 5 recipe (shared per-pair absmax, `input` tensors, 6 layers).

### `bitwidth_three_workloads.png` — bit-width across three workloads
Same temporal-difference split for **SDM / DiT / Fast-dLLM v2** under one shared
quantizer/classifier. Zero rate rises 46% → 79% → 80% (per-element); the annotation
notes DiT's **MAC-weighted** zero is 67.2% (weighting by layer compute, the number the
energy/speedup models use). Text diffusion is the most temporally sparse. The "% computed"
labels on top are the complement (work actually done = non-zero fraction).

### `fig13_roofline.png` — speedup & Defo-flip vs bandwidth (the main speedup deliverable)
The paper does **not** publish Ditto's DRAM, so speedup is a **bandwidth roofline**, not
a single point. Top panel: Ditto+Defo vs difference-only speedup over ITC as bandwidth
sweeps 2⁴–2¹² B/cycle; the **compute ceiling 8.89x** (attention+VPU-aware) is the
horizontal asymptote, and Ditto **reaches the paper's 1.5x at ~251 GB/s** (single-HBM2
class), marked with a star. Bottom panel: the Defo flip fraction falls 100% → ~14% as
bandwidth grows, meeting the paper's **14.4%** flip — which reverse-estimates the
paper's DRAM to ~3.2 TB/s. Paper 1.5x and 14.4% are overlaid reference lines.

### `fig16_defo_rescue.png` — Defo rescues memory-bound difference (paper Fig 16)
Difference-only execution dips to **0.67x** (below the ITC 1.0x line, shaded red — it is
*slower* than the baseline under tight memory, the paper's Fig 16 point), while
**Ditto+Defo holds ≥ 1.0x** by flipping memory-bound layers back to activation
execution. The x-axis is the analytical model's bandwidth knob (labeled as such — not a
measured physical bandwidth). Paper 1.5x target overlaid.

### `fig13_sweep.png` — what drives the memory ratio and the speedup (two swept knobs)
Two panels showing the model is not a single hidden assumption but an explicit sweep.
**Left:** the diff/act memory-access ratio vs on-chip buffer size — it falls from ~1.49x
toward the no-reload floor 1.34x as the buffer grows; the paper's **2.75x (Fig 8)** is
the reference line, reached only when buffer spilling dominates (i.e. the ratio is
buffer-driven). **Right:** Ditto+Defo speedup and Defo-flip% vs `BYTES_PER_CYCLE`, with
the paper 1.5x / ~14.4% lines — the same roofline behavior from a second knob. This is
the support behind §2's memory-ratio discussion.

### `fig13_energy.png` — SDM six-segment energy (paper Fig 13)
Energy normalized to ITC, broken into the paper's six segments
(**DRAM / SRAM / CORE / VPU / EU / DEFO**). **ITC = 1.00** (core-dominated, matching the
paper's core-heavy bar); **Cam-D = 1.34** — an *inversion* above ITC, its large green
DRAM segment is the previous-frame re-read with no compute saving; **Ditto+Defo = 0.66**
(saves 34%). SRAM per-access energy is **CACTI-7.0-measured** (818 pJ/64 B); EU/VPU/Defo
are modeled (labeled in the subtitle). The paper's 17.74% saving falls inside the
buffer-dependent sweep, noted on the bar.

### `fig13_energy_three.png` — six-segment energy across three workloads
The same six-segment breakdown run **unchanged** on SDM / DiT / Fast-dLLM v2, each using
its *own* measured bit-width (zero 45.9% / 67.2% / 80.3%). Ditto bars **0.66 / 0.54 /
0.30**; Cam-D inversion **1.34 / 1.14 / 1.04** (the inversion shrinks as the workload
becomes more linear/compute-dominated and less memory-bound). Same shape as the paper's
Fig 13 (ITC core-dominated, Cam-D DRAM blow-up, Ditto lowest) on every workload — the
model generalizes.

### `three_workload_trend.png` — the extendability headline (SDM → DiT → Fast-dLLM)
Two panels making the monotone trend explicit. **Left:** Ditto energy saving (bars
**34 → 46 → 70%**) rising while Cam-D inversion (line **1.34 → 1.14 → 1.04x**) shrinks.
**Right:** temporal-difference zero rate (bars **46 → 67 → 80%**) rising while the
attention MAC fraction (line **15.7 → 3.6 → 0.8%**) falls. Together: the more
linear-dominated and temporally sparse the workload, the larger Ditto's benefit —
strongest for large text diffusion. (Theoretical ceilings 8.9 / 16.5 / 28.4x in the
title are bandwidth→∞ limits, **not attainable speedups**.)

## RTL figure

### `rtl_fmax.png` — synthesis Fmax, and the bottleneck lesson
Measured Fmax (Vivado 2025.1, ZU3EG, out-of-context) of four PE variants. The three
carry-propagate PEs cluster at **~600 MHz** (`pe_diff` 599.9, `pe_diff_pipe` 575.7,
`pe_diff_slot` 612.4) — the timing report shows their common critical path is the
**32-bit accumulator carry chain**, so the 3-stage pipeline (which split the multiply/sum
logic, not the accumulator) gave *nothing*. The **carry-save accumulator** (`pe_diff_csa`)
removes the per-cycle carry propagation and reaches **3021 MHz — 5x** — while staying
functionally identical (resolved `acc` == numpy). The lesson: profile the critical path
before optimizing. (3021 MHz is an OOC estimate; place-and-route would lower it. See
[../rtl/SYNTHESIS.md](../rtl/SYNTHESIS.md).)
